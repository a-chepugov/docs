# Текст

## Заголовки (h1, h2, h3, h4, h5, h6)

```markdown
# Заголовок первого уровня
## Заголовок второго уровня
### Заголовок третьего уровня
#### Заголовок четвертого уровня
##### Заголовок пятого уровня
###### Заголовок шестого уровня
```

### Альтернативный синтаксис (для первых двух уровней)

```markdown
Заголовок первого уровня
========================

Заголовок второго уровня
------------------------
```

## Абзац (p)

Для отделения абзацев друг от друга их нужно разделить пустой строкой (два перевода строки)

```markdown
Это первый абзац.
Это все еще первый абзац.

Это второй абзац.

Это третий абзац.
Это опять третий абзац.
```

### Перенос строки (br)
Чтобы добавить перенос строки (без создания нового абзаца), надо в конце строки вставить два пробела

```markdown
переноса строки внутри··
абзаца
```

## Стилизация текста

### *курсив* (em)
```markdown
*любой текст* или _любой текст_
```

### **жирный** (strong)
```markdown
**любой текст** или __любой текст__
```

- выделение и курсив:
```markdown
`***любой текст***`
```

- вложенные выделение и курсив:
```markdown
`**любой _элемент_ текста**`
`_любой **элемент** текста_`

```

- зачеркивание:
```markdown
~~любой текст~~
```

## Списки

### Ненумерованный
В начале строки поставить один из знаков: `*`, `-`, `+`

```markdown
* элемент ненумерованного списка
- ещё один элемент ненумерованного списка
+ метки элементов могут быть разными
```

### Ненумерованный
В начале строки поставить любое число и точку.
Число в первой строке определяет начало отсчета

```markdown
3. Элемент №2 того же списка
1. Элемент нумерованного списка
3. Элемент №3 того же списка
9. Элемент №4 списка — элементы нумеруются по порядку, числа и их порядок в начале строки не имеет значения
```

# Горизонтальная черта (hr)

в начале строки поставить 3 или больше символа: `***`, `---`, `___`

```markdown
текст до
---
текст после
```

# Ссылки

## строчные

```markdown
[Видимый текст ссылки](http://example.com/ "Необязательный всплывающий заголовок")
```

## ссылки с выделенными определением и обращением

Позволяет использовать определение одной ссылки несколько раз

```markdown
Окружающий текст [ссылка 1] окружающий текст.
Повторное использование [ссылка 1].
Окружающий текст [альтернативная подпись ссылки 1][ссылка 1] окружающий текст.

Ссылка на [второй][ссылка 2] также [третий][ссылка 3] ресурсы.

[ссылка 1]: http://example.com/ "Необязательный всплывающий заголовок 1"
[ссылка 2]: http://example.net/ 'Необязательный всплывающий заголовок 2'
[ссылка 3]: http://example.edu/ (Необязательный всплывающий заголовок 3)
```

В документе отображается только обращение к ссылке, а определение будет скрыто

## "быстрые ссылки"
чтобы быстро создать ссылку на URL или адрес электронной почты, заключите их в угловые скобки.

```markdown
<https://example.org/>
<admin@example.com>
````

# Нестандартные расширения

## Использование HTML

## надстрочный и подстрочный текст:
```markdown
Это <sub>подстрочный </sub> и <sup>надстрочный</sup> текст
```

## специальное форматирование
```markdown
<kbd>CTRL</kbd> + <kbd>P</kbd>
```
  
## Таблицы
```markdown
| Заголовок1 | Заголовок2 | Заголовок3 |
| ---------- | ---------- | ---------- |
| текст      | текст      | текст      |
|            |            |            |
| текст      |            |            |
|            |            | текст      |
```

Выравнивание (количество пробелов), наличие текста в ячейках и количество символов `-` (в строке отделения заголовка и данных) не имеют значения, и важны только для чтения без предобработки


### Выравнивание ячеек
```markdown
| по левому краю | по центру | по правому краю |
| :------------- | :-------: | --------------: |
| текст          |   текст   |           текст |
```

## Списки с метками
```markdown
- [x] Задача 1
- [ ] Задача 2
- [ ] Задача 3
```

## Ссылки на элементы страницы
```markdown
[Перейти к метке 1](#mark1)
Какой-то контент
<a id="mark1">метка 1</a>
```

## Сноски
```markdown
Любой текст[^1]. Любой текст[^2][^3].

[^1]: Текст сноски 1
[^2]: Текст сноски 2
[^3]: [википедия](https://ru.wikipedia.org/)
```

### Комментарии

Для вставки комментария, который не отобразится в конечном документе можно использовать такие конструкции:

```markdown
[comment]: <> (Это комментарий, который не будет включен в вывод)

[//]: # (Это может быть наиболее платформонезависимый комментарий)
```

## Ссылки на файлы
```markdown
[отображаемый текст](путь к файлу)
```
Путь к файлу может указываться в абсолютной или относительный форме

Пример
:
```markdown
[документация](./Markdown.md)
```

## Метки оповещений
```markdown
> [!NOTE]
> Полезная информация

> [!TIP]
> Совет

> [!IMPORTANT]
> Важная информация

> [!WARNING]
> Срочно оповещение

> [!CAUTION]
> Критическое оповещение
```
